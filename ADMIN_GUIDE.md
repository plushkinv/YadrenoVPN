# Настройка и администрирование бота

В этом разделе описаны основные возможности для пользователей и администраторов Telegram-бота Yadreno VPN.

---

## Ручная установка и запуск (Linux/Ubuntu)

> ℹ️ Если вы предпочитаете автоматическую установку — воспользуйтесь скриптом, как описано в [README](README.MD). Этот раздел — для тех, кто любит всё контролировать вручную или если автоматический режим не сработал.

### Шаг 1: Подготовка системы
Обновите пакеты и установите необходимые зависимости:
```bash
sudo apt update && sudo apt install -y python3-venv python3-pip git
```

### Шаг 2: Клонирование репозитория
Скачайте проект с GitHub:
```bash
git clone https://github.com/plushkinv/YadrenoVPN.git
cd YadrenoVPN
```

### Шаг 3: Создание виртуального окружения и установка зависимостей
```bash
python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
deactivate
```

### Шаг 4: Настройка
1. Скопируйте файл шаблона настроек:
   ```bash
   cp config.py.example config.py
   ```
2. Отредактируйте `config.py`, указав ваши данные:
   - `BOT_TOKEN`: Токен вашего бота от @BotFather.
   - `ADMIN_IDS`: Ваш Telegram ID.
   - `TELEGRAM_API_URL` *(необязательно)*: Адрес Telegram Bot API. По умолчанию — стандартный `https://api.telegram.org`. Можно указать один кастомный адрес (прокси/зеркало), если доступ к стандартному API ограничен. Если нужно несколько резервных адресов, укажите их списком: `["https://proxy-1.example.com", "https://proxy-2.example.com", "https://api.telegram.org"]`. Бот начнет с первого адреса и переключится на следующий только если текущий адрес не ответит. После последнего адреса снова используется первый. Если параметр не указан в конфиге — бот автоматически использует стандартный адрес.

> [!IMPORTANT]
> Файл `config.py` содержит секретные данные, поэтому он автоматически игнорируется Git и не попадет в публичный репозиторий. Всегда используйте `config.py.example` как образец для новых установок.

### Шаг 5: Запуск
#### Способ 1: Обычный запуск (для тестов)
```bash
source venv/bin/activate
python3 main.py
```

> ℹ️ **Примечание:** При старте бот проверит версию базы данных и выведет в лог информацию о статусе миграций (успех/ошибка).

#### Способ 2: Автозапуск через systemd (рекомендуется)
Чтобы бот работал постоянно и запускался сам после перезагрузки сервера:

1. Копируем файл сервиса:
   ```bash
   cp yadreno-vpn.service /etc/systemd/system/
   ```
2. Активируем и запускаем:
   ```bash
   systemctl daemon-reload
   systemctl enable yadreno-vpn
   systemctl start yadreno-vpn
   ```

> ℹ️ Файл `yadreno-vpn.service` уже настроен на использование виртуального окружения (`/root/YadrenoVPN/venv/bin/python`).

## Ручное обновление бота

### Через команду в боте
Наберите команду в боте:
```
/update
```

### Через консоль сервера
Если команда `/update` не помогла, подключитесь к серверу по SSH и выполните:
```bash
cd /root/YadrenoVPN
git pull
systemctl restart yadreno-vpn
```

Если обновление зависимостей:
```bash
cd /root/YadrenoVPN
source venv/bin/activate
pip install -r requirements.txt
deactivate
systemctl restart yadreno-vpn
```

### 📊 Информация о ключах
- Статус подключения (активен/истек)
- Использованный трафик
- Дата окончания подписки
- Сервер и протокол подключения
- История платежей

В режиме подписок пользователь получает одну ссылку, через которую автоматически подтягиваются все доступные протоколы сервера. При продлении срок, тариф, лимит трафика и лимит устройств обновляются автоматически по выбранному тарифу.

## Для администраторов

Администраторы имеют доступ к дополнительным функциям через кнопку "⚙️ Админ-панель":
- Управление серверами
- Настройка тарифов (включая лимиты трафика)
- Управление пользователями (поиск по Telegram ID, @username или e-mail из панели 3X-UI)
- В карточке пользователя показывается шаблон e-mail для поиска в панели, в карточке конкретного ключа — его полный e-mail на панели
- Блокировка пользователя отключает его VPN-доступ на панели 3X-UI. Разблокировка синхронизирует ключи обратно и включает только действующие по сроку и трафику ключи.
- **Управление пробной подпиской** (вкл/выкл, выбор тарифа)
- **Обновление бота** (обычные обновления, блокирующие и бета-версии)

- Редактирование текстов бота (главная, справка, ссылки)
- 🤖 **Yadreno Admin** — чат с агентом для управления сервером и ботом
- 💚 **Поддержка автора** (реквизиты разработчика скрипта и ссылки)
- Рассылка сообщений
- **Скачивание, очистка и анализ логов** (скачивание полного лога работы бота, лога со списком ошибок, отправка `bot.log` в Yadreno Admin для анализа, либо полная очистка текущих и старых ротированных логов. Размер одного лог-файла ограничен 1 МБ, хранится 3 резервных копии)
- Просмотр статистики

### 📢 Рассылка сообщений

В разделе **Админ-меню** → **Рассылка** можно подготовить сообщение для пользователей, посмотреть превью, выбрать фильтр получателей и запустить отправку.

Когда рассылка запущена, в этом же разделе появляется кнопка **🛑 Остановить рассылку**. Она мягко прерывает текущую отправку: уже начатое сообщение будет обработано, а новые сообщения после остановки отправляться не будут. После остановки бот покажет итог с количеством отправленных сообщений, ошибок и неотправленных получателей.

Если рассылка была прервана по техническим причинам и раздел всё ещё показывает, что она идёт, нажмите **🛑 Остановить рассылку** — бот сбросит зависшее состояние и снова разрешит запуск новой рассылки.

### 🖥️ Поддержка версий 3X-UI

Бот поддерживает панели 3X-UI версий v2.x, v3.0.x, v3.1.x-v3.2.x и v3.3+. При подключении бот сам определяет версию панели и настраивается на работу с ней — никаких ручных действий не требуется.

Версия панели, статус подключения и время последней проверки показываются в карточке каждого сервера в разделе **Сервера**.

После обновления панели 3X-UI никаких дополнительных действий обычно не требуется — бот автоматически перенастроится при следующем подключении. В 3X-UI v3.3+ настройки панели переехали в новый API-namespace; бот поддерживает и старый, и новый путь.

Если на одном сервере используются разные протоколы, например Hysteria2 и VLESS Reality, порядок inbound в панели не важен. Бот сам выставляет нужный режим подключения для VLESS Reality при выдаче ключа и при синхронизации с панелью.

Если после обновления панели пользователь видит ошибку `Не удалось получить subscription URL`, проверьте в 3X-UI раздел **Settings → Subscription**:
- подписка включена (`Enable`);
- корректно заданы `subDomain`, `subPort`, `subPath` или готовый `subURI`;
- порт подписки доступен снаружи и не совпадает с портом веб-панели.

### 🤖 Yadreno Admin

В главном меню админ-панели есть кнопка **«🤖 Yadreno Admin»** — это чат с ИИ-агентом, который может управлять вашим сервером и ботом.

Примеры того, что можно делать с помощью агента, смотрите в видео: https://www.youtube.com/watch?v=ACPu03aAJns

Если у вас возникли вопросы по YadrenoVPN, вы всегда можете задать их в [@YadrenoAdmin_Bot](https://t.me/YadrenoAdmin_Bot) — там вас бесплатно проконсультируют и ответят на любые вопросы.

**Начало работы:**
1. При первом входе нажмите **«🔑 Указать api_key»**.
2. Получить ключ можно в [@YadrenoAdmin_Bot](https://t.me/YadrenoAdmin_Bot) в разделе **«Профиль»**.
3. После сохранения ключа пишите задачи обычными сообщениями — например, «обнови панель 3X-UI» или «покажи логи».

> Один `api_key` действует на весь бот. IP сервера определяется автоматически при сохранении ключа. Обычный чат Yadreno Admin и `/yaa` работают в разных сессиях хаба и могут выполняться параллельно. Активные запросы сохраняются в базе бота, поэтому кнопка «🔄 Ну чё там?» может восстановить ожидание ответа даже после перезапуска VPN-бота.

**Кнопки управления:**

| Кнопка | Назначение |
|--------|------------|
| **🆕 Новый чат** | Сбрасывает контекст диалога — начинает разговор с чистого листа. Доступна, когда агент не занят. |
| **🔑 Заменить api_key** | Позволяет указать или сменить ключ доступа |
| **❌ Отмена** | Останавливает текущую задачу агента (то же самое, что `/cancel`) |
| **🔄 Ну чё там?** | Показывает последний ответ или статус агента. Если бот перезапускался во время работы агента, восстанавливает ожидание результата по сохранённому запросу |

**Что умеет агент:**
- Выполнять команды на сервере
- Работать с файлами
- Выполнять SQL-запросы к базе данных бота
- Анализировать логи и состояние сервера
- Анализировать приложенные фото и поддерживаемые файлы; видео/GIF можно ставить как медиа страницы, но не анализировать

Фото и поддерживаемые документы до 10 МБ можно отправлять прямо в обычный чат Yadreno Admin: бот скачивает файл локально, загружает его в хаб и показывает ответ агента в том же диалоге. Видео и GIF не скачиваются и не анализируются; если добавить к ним подпись с задачей, агент получит Telegram `file_id` и сможет поставить это вложение как медиа страницы.

### `/yaa` — редактирование страниц прямо из бота

Откройте любую пользовательскую страницу и отправьте команду:

`/yaa сделай кнопку поддержки зелёной`

Агент поймёт, какая страница открыта, и внесёт изменения. Если страница изменилась — бот сразу покажет обновлённый вариант. `/yaa` использует отдельную сессию от обычного чата Yadreno Admin, поэтому запрос на редактирование страницы не мешает текущему диалогу с агентом.

При вызове `/yaa` агент получает сохранённое состояние открытой страницы и список видимых кнопок текущего экрана. Это помогает ему быстрее менять текст, медиа или статические кнопки без лишних запросов.

**На каких страницах работает:** Главная, Справка, Пробная подписка, Экран перед оплатой, Выбор оплаты при продлении, Реферальная система, Выдача ключа, Мои ключи, карточка ключа, замена ключа, переименование ключа и настройка ключа после оплаты.

**Что можно менять через `/yaa`:**
- Текст страницы
- Медиа страницы: фото, видео или GIF (приложите файл к команде `/yaa`; агент получает Telegram `file_id` и может поставить его на открытую страницу. Видео/GIF передаются без скачивания и анализа)
- Статические кнопки (названия, цвета и кастомные эмодзи-иконки)
- Скрытый часовой пояс отображения дат во всём боте. Примеры: `/yaa установи часовой пояс Москва`, `/yaa поставь часовой пояс Europe/Moscow`, `/yaa показывай даты в UTC+3`.

Форматирование передаётся агенту без потерь. Можно выделять текст средствами Telegram и вставлять Premium-эмодзи прямо в команду, например: `/yaa поставь [кастомный эмодзи] на кнопку «Справка»`. Для отправки кастомных эмодзи пользователям владелец бота должен иметь Telegram Premium.

Перед каждым запросом `/yaa` бот автоматически создаёт резервную копию своей базы. Простые изменения выполняются без лишнего плана агента. Если запрос сложный и не относится к открытой странице, агент предложит перейти в обычный чат Yadreno Admin; задачи сразу по нескольким серверам следует выполнять через основного бота Yadreno Admin.

> ⚠️ Кнопки конкретных ключей, серверов, протоколов и тарифов генерируются ботом автоматически и не меняются через `/yaa`.

### 🕒 Часовой пояс дат

База данных хранит даты в UTC, а бот показывает их в часовом поясе из скрытой настройки `display_timezone`. По умолчанию используется Москва (`Europe/Moscow`). Настройка влияет на даты регистрации, сроки ключей и историю оплат у администратора и пользователя.

### 💾 Система бэкапов

Бот автоматически создаёт резервные копии всех баз данных ежедневно в 03:05.

**Что сохраняется:**
- `vpn_bot.db` — база данных бота (пользователи, ключи, платежи, настройки)
- `server_НАЗВАНИЕ_x-ui.db` — база каждого активного VPN-сервера

База бота копируется согласованно даже во время работы бота. Это важно для стабильных бэкапов: не нужно вручную копировать рабочий файл базы во время запущенного бота, используйте автоматические архивы или локальные копии из папки `backup/`.

**Два способа хранения:**

| Способ | Описание |
|--------|----------|
| **📨 Telegram** | ZIP-архив `backup_YYYY-MM-DD.zip` отправляется всем администраторам |
| **📂 Локально** | Неархивированные `.db` файлы сохраняются в папку `backup/YYYY-MM-DD/` в корне проекта |

**Локальное хранение:**
- Файлы хранятся **без архивации** — к ним можно обращаться напрямую
- Любые файлы и ссылки внутри `backup/` старше **7 дней** удаляются автоматически независимо от имени и расширения; опустевшие подпапки также очищаются
- Папка `backup/` не попадает в репозиторий

### 🔄 Система обновлений

В разделе **Настройки бота** -> **Обновления** администратор может обновлять код бота прямо из GitHub репозитория.

Виды обновлений:
1. **Обычные обновления** — применяются по нажатию кнопки «Обновить и перезапустить». О выходе таких обновлений бот автоматически уведомляет администраторов (если включён планировщик).
2. **Блокирующие обновления (`!`)** — если сообщение коммита начинается с `!`, бот предупредит, что это важное обновление (например, требующее миграции БД). При нажатии «Обновить до блокирующего коммита» бот установит именно этот коммит.
3. **Бета-версии (`?`)** — если сообщения всех новых коммитов начинаются с `?`, бот **не будет** рассылать уведомления администраторам. Вы можете зайти в раздел обновлений и нажать специальную кнопку **«Накатить бета версию»**, чтобы протестировать новые функции на свой страх и риск.

Уведомления о выходе новых версий по умолчанию включены. Это скрытая настройка без отдельной кнопки в админке: чтобы отключить или снова включить уведомления, попросите об этом **Yadreno Admin**. Ручная проверка и установка обновлений в разделе **Обновления** останутся доступны.

Если что-то пошло не так, можно воспользоваться кнопкой **«Принудительно перезаписать»** — она скачает самую последнюю версию из ветки и затрёт все локальные изменения в коде (база данных при этом не затрагивается).

### 📊 Система учёта трафика

Бот поддерживает ограничение пользователей по объему трафика:
- **Установка лимитов**: При создании/редактировании тарифа вы можете указать лимит в ГБ. Если указать `0` — трафик будет безлимитным.
- **Продление другим тарифом**: При продлении ключ переходит на выбранный тариф. Лимит трафика заменяется лимитом нового тарифа, использованный трафик обнуляется, а значение `0` снимает старое ограничение и делает трафик безлимитным.
- **Остаток при переезде**: Если пользователь меняет сервер в боте, его неизрасходованный остаток трафика бережно переносится на новый сервер.
- **Ежемесячный сброс**: В разделе **Админ-меню** → **Оплаты** есть переключатель **«🔄 Автосброс трафика 1-го числа»**. Если он включен, то 1-го числа каждого месяца всем пользователям обнуляется израсходованный трафик, и им снова доступен полный лимит по их тарифу.
- **Уведомления**: Бот сам предупредит клиента, когда у того останется 10%, 5%, 3% и менее процентов трафика.

В режиме подписок на новых версиях 3X-UI бот учитывает общий расход клиента на конкретном сервере, даже если подписка содержит несколько протоколов. На старых панелях бот использует доступные счётчики панели по протоколам. Если у пользователя есть ключи или подписки на разных добавленных серверах, их трафик считается отдельно по каждому серверу.

### 🔔 Уведомления об оплатах

В разделе **Админ-меню** → **Оплаты** есть переключатель **«🔔 Сообщать об оплатах»**. Если он включён, при каждой успешной оплате все администраторы получают сообщение вида:

```
💰 Новая покупка

👤 Пользователь: 396437466 (@username)
🎫 Тариф: Light — 1 месяц
💳 Метод: ⭐ Telegram Stars
💵 Сумма: 100 ⭐
```

- ID пользователя — кликабельная ссылка на его Telegram-профиль
- Для продления заголовок будет «🔄 Продление»
- Строка «🌐 Хост» появляется только при продлении (когда сервер уже известен); для новых ключей она скрыта
- Поддерживаются все способы оплаты: Stars, Крипто, ЮКасса, WATA, Platega, Cardlink, Баланс, Пробная подписка
- Для одной успешной оплаты отправляется одно уведомление; повторная проверка уже обработанного платежа не создаёт дубль
- Уведомление не ломает процесс оплаты — если отправка не удалась, ошибка просто логируется

### 🔄 Синхронизация данных с панелью

Бот является **единственным источником правды** для дат истечения ключей и лимитов трафика. Данные на панели 3X-UI автоматически синхронизируются с базой бота.

**Автоматическая сверка (1-е число каждого месяца):**
Бот автоматически сравнивает данные на панелях с базой данных. Если обнаружены расхождения — бот исправляет данные на панели и отправляет отчёт администраторам. Уже актуальные записи не перезаписываются повторно.

**Ручная синхронизация:**
Перейдите: **Админ-меню** → **Пользователи**

| Кнопка | Описание |
|--------|----------|
| **📤 Выгрузить в панель (БД → Панель)** | Переносит даты истечения, лимиты трафика и статусы из бота на панели 3X-UI. Бот проверяет все активные ключи, но отправляет изменения только для устаревших записей. Используйте, если данные в боте верные, а на панели устарели. |
| **📥 Загрузить из панели (Панель → БД)** | Считывает данные с панелей 3X-UI и обновляет базу бота (даты, лимиты, трафик). Используйте, если данные на панели верные, а в боте устарели. |

После нажатия бот сразу покажет, что синхронизация запущена. Когда операция закончится, это же сообщение обновится итоговым отчётом по обработанным ключам, реально обновлённым записям, созданным клиентам, записям без изменений и ошибкам синхронизации. Если какая-то панель недоступна или не отвечает вовремя, ошибка будет отражена в отчёте.

При покупке, продлении и выдаче ключей бот автоматически повторяет обращение к панели при временных сбоях и проверяет, не был ли ключ уже создан до таймаута. Если панель полностью недоступна, оплата или изменение сохраняются в боте, а администратор может позже запустить **📤 Выгрузить в панель (БД → Панель)**, чтобы дожать выдачу или продление.

> 💡 Используйте **📤 Выгрузить**, если вы редактировали данные в боте и хотите применить их на панели. Используйте **📥 Загрузить**, если вы вносили изменения напрямую на панели 3X-UI и хотите синхронизировать их обратно в бот.

### 🔌 Скрытие inbound от бота

Если какой-то inbound в 3X-UI нужно временно исключить из выдачи клиентам, добавьте в начало его названия префикс `--!`.

Пример:
```text
--! Germany VLESS
```

После ближайшей синхронизации бот:
- перестанет показывать этот inbound в выборе протокола;
- не будет добавлять в него новых клиентов;
- уберёт из него клиентов, которыми управляет бот.

Чтобы вернуть inbound в работу, удалите `--!` из начала названия и запустите синхронизацию. В режиме подписок бот снова добавит клиентов в этот inbound. В режиме одиночных ключей бот восстановит ключ только в том inbound, к которому он был привязан раньше.

### 📵 Пользователи, заблокировавшие бота

Если при ручной рассылке или автоуведомлении об истечении ключа Telegram сообщает, что пользователь заблокировал бота, бот автоматически помечает такого пользователя в базе. После этого ему больше не отправляются ручные рассылки и уведомления об истечении ключей.

В разделе **Админ-меню** → **Пользователи**:
- в статистике есть строка **📵 Заблокировали бота**;
- в списке пользователей есть фильтр **📵 Бот заблокирован**;
- в карточке пользователя показывается статус доставки.

Флаг снимается автоматически, когда пользователь снова пишет боту или нажимает любую кнопку. Обычные ошибки отправки не ставят этот флаг.

Этот статус не отключает VPN-доступ: он означает только, что Telegram не принимает сообщения от бота. VPN-клиенты отключаются на панели только при ручной блокировке пользователя администратором.

### 🗑️ Удаление ключа

В карточке пользователя можно зайти в любой ключ и нажать **🗑️ Удалить ключ**. Перед удалением будет показано предупреждение:

- Ключ удаляется безвозвратно с панели 3X-UI и из базы данных
- Пользователь не сможет подключиться, заменить или восстановить ключ
- Средства не возвращаются автоматически — компенсация на админе

> ⚠️ Используйте эту функцию осторожно. Если пользователь оплатил ключ, вам нужно самостоятельно компенсировать ему (пересоздать ключ, пополнить баланс и т.д.).

### 🗑️ Синхронизация удалённых ключей

Перейдите: **Админ-меню** → **Пользователи** → **🗑️ Синхронизация удалённых**

| Кнопка | Описание |
|--------|----------|
| **🧹 Очистить панель** | Удаляет с панели 3X-UI всех клиентов бота, которых нет в базе данных. Если какие-то серверы недоступны — бот предупредит в отчёте. |
| **🗑️ Очистить базу** | Сканирует все серверы и показывает отчёт по категориям (см. ниже). Ничего не удаляется автоматически — вы сами выбираете, что удалить. |

#### Категории отчёта «Очистить базу»

После нажатия «Очистить базу» бот сканирует все серверы (включая деактивированные) и показывает отчёт по категориям:

| Категория | Что это означает | Рекомендуемое действие |
|-----------|---------|---------------|
| 📋 Ключи без сервера | Для ключа в БД не указан сервер. Сюда попадают новые купленные ключи, которые ещё не были настроены. | Можно удалять только если уверены, что это мусорные записи. |
| 👻 Удалённые серверы | Ключ привязан к серверу, который был полностью удалён из настроек бота. | Можно удалять, так как ключ всё равно не заработает. |
| 🟢 Отсутствуют на панели | Сервер доступен, но конкретного клиента на нём нет (удалён вручную через 3X-UI). | Требуется подтверждение и повторная проверка перед удалением из БД. |
| 🔴 / ⏸️ Недоступный сервер | Сервер деактивирован в боте или просто не отвечает на запросы. | **Осторожно!** Если сервер просто временно упал, удаление ключа из базы лишит пользователя доступа даже после починки сервера. |

> 🚨 **Важно:** Принудительное удаление используйте только если сервер утерян безвозвратно.

> 🚨 **Важно (Недоступные серверы):** Если сервер недоступен (упал, РКН, перезагрузка), кнопка удаления будет сопровождаться предупреждением. Используйте принудительное удаление только если уверены, что сервер утерян безвозвратно!

### 🔗 Реферальная система

### 📂 Группы тарифов и серверов

Механизм групп позволяет изолировать серверы и тарифы друг от друга. По умолчанию все элементы попадают в группу "Основная".

**Основные принципы:**
- **Создание группы**: В Админ-меню → Оплаты → Группы тарифов можно добавить новую группу.
- **Привязка**: При создании нового тарифа или сервера бот предложит выбрать, к какой из групп они будут привязаны.
- **Изоляция**: Пользователи, купившие ключ с тарифом из определенной группы, смогут продлевать его только тарифами из этой же группы. При замене сервера им будут доступны только серверы из их группы.
- **Отображение**: Если создана больше одной группы (и в ней есть хотя бы 1 активный тариф и 1 сервер), они будут визуально разделены на категории в меню покупки для пользователей.

В админ-панели можно менять тексты бота без перезагрузки.

При нажатии на кнопку редактирования показывается **превью** — сообщение ровно в том виде, как его увидит пользователь. Для сохранения достаточно отправить новый текст или медиа с подписью в чат — превью обновится мгновенно.

**Поддержка медиа**: Можно отправить фото, видео или GIF с подписью — оно сохранится и будет показываться пользователям. Если у текста уже есть медиа, отправка нового текста меняет только текст и сохраняет медиа. Чтобы убрать медиа и оставить только текст, нажмите `🗑 Удалить медиа`.

**Лимиты**: Обычный текст — до 4096 символов, подпись к медиа — до 1024 символов. Если текст длиннее — бот обрежет до лимита.

**Плейсхолдеры**: В некоторых текстах поддерживаются переменные (например, `%тарифы%`, `%ключ%`). Они автоматически заменяются на актуальные данные при показе пользователю. Регистр букв не важен: `%ключ%`, `%КЛЮЧ%` и `%Ключ%` сработают одинаково.

**Доступные тексты:**
1. **Текст главной страницы** (`⚙ Настройки бота` → `✏️ Редактирование текстов`): Приветствие при `/start`. Плейсхолдеры: `%тарифы%` (вставка списка тарифов в это место), `%без_тарифов%` (скрыть список тарифов).
2. **Текст справки**: Содержимое раздела `/help`.
3. **Текст перед оплатой**: Условия, которые показываются при нажатии «Купить ключ». (Плейсхолдеры в системе оплат не поддерживаются).
4. **Текст пробной подписки** (`Пробная подписка` → `✏️ Изменить текст`): Описание и условия триала.
5. **Реферальная страница** (`⚙ Настройки бота` → `✏️ Редактирование текстов` → `Реферальная страница`). Плейсхолдеры: `%ссылка%` (реферальная ссылка пользователя), `%статистика%` (статистика по уровням и баланс).
6. **Текст уведомления об истечении** (`Рассылка` → `📨 Уведомления` → `✏️ Редактировать текст`). Плейсхолдеры: `%дней%`, `%имяключа%`.
7. **Текст выдачи ключа** (`⚙ Настройки бота` → `✏️ Редактирование текстов`). Плейсхолдеры: `%ключ%` — ключ в моноширинном виде для копирования, `%ссылка%` — кликабельная ссылка. Можно использовать один из них или оба сразу.
8. **Страницы «Мои ключи»** редактируются через `/yaa` с открытого пользовательского экрана. Основная страница поддерживает `%списокключей%`; пустая страница по умолчанию показывает кнопку «Купить ключ».
9. **Страницы ключей** редактируются через `/yaa` с открытого пользовательского экрана: карточка ключа, экран показа ненастроенного ключа, недоступное продление, шаги замены ключа, переименование ключа и выбор сервера/протокола после оплаты.

**Ссылки:**
- Настройка URL для кнопок «Новости» и «Поддержка» (`✏️ Редактирование текстов` → соответствующий раздел). Можно менять URL, название кнопки или полностью скрыть её.
- Кнопки главной страницы «Мои ключи» и «Купить ключ» отображаются обычным стилем Telegram. Если администратор меняет цвет кнопок через `/yaa`, это сохраняется и не перезаписывается обновлениями.
- Названия кнопок способов оплаты на экранах покупки и продления меняются через `/yaa` прямо на открытом экране; включение/выключение самих методов оплаты настраивается в разделе **Оплаты**.
- Кнопки, которые зависят от конкретного ключа, сервера, протокола или тарифа, бот генерирует автоматически. Через `/yaa` меняются только текст, медиа и статические кнопки страницы.

> 💡 **Как редактировать**: Нажмите кнопку редактирования → увидите превью → отправьте новый текст или медиа с подписью → сохранение мгновенное. Если нужно убрать медиа, нажмите `🗑 Удалить медиа`.

#### Специальные теги для текста главной страницы

В тексте главной страницы можно использовать теги для управления выводом списка тарифов:

| Тег | Поведение |
|-----|-----------|
| *(тег не указан)* | Список активных тарифов **автоматически добавляется в конец** сообщения |
| `%тарифы%` | Список тарифов вставляется **в указанное место** текста |
| `%без_тарифов%` | Тарифы **не выводятся** совсем (тег удаляется из текста) |

> 💡 **Пример:** Чтобы разместить тарифы между двумя абзацами, вставьте `%тарифы%` в нужное место. Если тарифы вообще не нужны на главной — добавьте `%без_тарифов%` в любом месте текста.
